# ESLint 규칙 설명

## MSA 지향 모듈 구조 규칙

### Rule 1: 플랫폼 독립 코드에서 next/* import 금지
- **대상**: `modules/*/model`, `modules/*/api`, `modules/*/lib`
- **규칙**: `next/*` import 사용 금지
- **이유**: 이 폴더들은 플랫폼 독립 코드로, 향후 Mobile(React Native)에서도 재사용 가능해야 합니다.
- **예외**: `modules/*/ui`는 web 전용이므로 `next/*` 사용 가능

### Rule 2: 외부에서 modules 내부 직접 import 금지
- **대상**: `app/**`, `shared/**`
- **규칙**: `modules/<domain>/api`, `model`, `lib`, `ui` 직접 import 금지
- **허용**: `modules/<domain>/index.ts`만 import 가능
- **이유**: MSA 경계 유지 및 캡슐화

### Rule 3: modules 간 직접 참조 금지 (경고)
- **대상**: `modules/**`
- **규칙**: 다른 도메인의 내부를 절대 경로로 직접 import하는 것 경고
- **허용**: 같은 도메인 내부는 상대 경로(`../model`) 사용 가능
- **이유**: 도메인 간 결합도 최소화

## 예시

### ✅ 올바른 사용
```typescript
// app/page.tsx
import { ExamYearPage } from "@/modules/exam"; // ✅

// modules/exam/ui/TestModePage.tsx
import { useExamContext } from "../model"; // ✅ 같은 도메인 내부
import { useRouter } from "next/navigation"; // ✅ ui는 web 전용
```

### ❌ 잘못된 사용
```typescript
// app/page.tsx
import { ExamYearPage } from "@/modules/exam/ui"; // ❌

// modules/exam/api/subject.ts
import { useRouter } from "next/navigation"; // ❌

// modules/exam/ui/TestModePage.tsx
import { useAuthContext } from "@/modules/auth/model"; // ⚠️ 다른 도메인 직접 참조
```
