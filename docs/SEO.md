# 검색엔진최적화(SEO) 가이드

구글·네이버 검색 노출을 위한 전략과 체크리스트를 정리한 문서입니다.

---

## 1. 네이버 SEO – 효과 높이는 방법

### 1.1 내부링크

- **2~3개의 내부링크**를 넣어 관련 페이지로 연결하면 **체류시간**이 늘어납니다.
- 사용자가 다음에 볼 만한 콘텐츠(같은 주제, 시리즈, 관련 서비스)로 자연스럽게 유도하세요.

### 1.2 글 하단 요약·재방문 유도

- 글 **하단에 요약 문단**을 두어 핵심을 다시 정리합니다.
- “다른 글도 보기”, “다음 시리즈 보기” 등 **재방문·이탈 방지** 문구를 넣습니다.

### 1.3 시리즈·연속 콘텐츠

- **동일 주제의 시리즈**를 연속으로 발행하면 네이버가 주제 신뢰도를 쌓을 수 있습니다.
- 시리즈 목차·이전/다음 글 링크로 내부링크를 강화하세요.

---

## 2. 구글 SEO – 효과 높이는 방법

### 2.1 출처가 명확한 데이터

- 통계, 수치, 사실 주장에는 **출처(원문 링크, 기관명)**를 명시합니다.
- 출처가 있는 콘텐츠는 E-E-A(경험·전문성·권위성·신뢰성)에 유리합니다.

### 2.2 소제목(H2, H3)으로 논리 구성

- **문단마다 소제목(H2, H3)**을 사용해 구조를 명확히 합니다.
- 구글은 제목 계층을 통해 주제·흐름을 파악하므로, 논리적인 구조가 검색 품질에 도움이 됩니다.

### 2.3 신뢰도 높은 외부링크

- **신뢰할 수 있는 외부 사이트**(공식 기관, 학계, 공인 자료)로 링크를 걸면 검색 품질·신뢰도 평가에 긍정적입니다.
- 필요 시 `nofollow`를 사용해도, 참고 자료로의 연결은 가치가 있습니다.

## 3. SEO 최적화 공통 체크리스트

| 항목 | 설명 |
|------|------|
| **제목에 주요 키워드 포함** | `<title>`, `<h1>`에 검색 대상 키워드를 자연스럽게 넣기 |
| **본문 내 자연스러운 키워드 분포** | 키워드 스터핑 없이 본문·소제목에 균형 있게 사용 |
| **소제목 구조(H2, H3) 반영** | 한 페이지에 H1 1개, 이하 H2 → H3 순서로 계층 유지 |
| **이미지 캡션과 내부링크 삽입** | 이미지에는 `alt` + 필요 시 캡션, 본문에 관련 페이지 내부링크 2~3개 |
| **결론에 요약 및 행동 유도** | 글 마지막에 요약 문단 + “다음 단계”(다른 글 보기, 서비스 이용 등) 문장 추가 |

## 4. 구글 서치 콘솔 – 사이트맵

### 4.1 "Sitemap이 HTML입니다" 오류 해결

**원인**  
서치 콘솔에 **일반 페이지 URL**을 사이트맵으로 넣으면, 해당 URL은 HTML 페이지이기 때문에 구글이 "Sitemap이 HTML입니다"라고 표시합니다.

- ❌ **제출하면 안 되는 예**: `/exam/536/year`, `/exam/538/year`, `/exam/536/3165/memorize-mode`  
  → 이 주소들은 **페이지 URL**이지, 사이트맵 파일이 아닙니다.
- ✅ **제출할 URL (1개만)**  
  - `sitemap.xml`  
  - 또는 전체 주소: `https://www.qknou.kr/sitemap.xml`

**조치**  
1. 서치 콘솔 → **사이트맵** 메뉴에서, 잘못 등록한 **페이지 URL 4개는 제거**하거나 무시합니다.  
2. **새 사이트맵 추가** 입력란에 **`sitemap.xml`** 만 입력하고 **제출**합니다.  
3. 구글이 `/sitemap.xml`을 읽으면 XML 형식의 사이트맵을 받게 되고, "발견된 페이지"가 0이 아닌 값으로 올라갑니다.

### 4.2 적용된 수정(코드)

| 위치 | 내용 |
|------|------|
| **next.config.ts** | `/sitemap` → `/sitemap.xml` 301 리다이렉트. 잘못된 URL로 요청해도 항상 XML이 나가도록 함. |
| **src/app/sitemap.ts** | `dynamic = "force-dynamic"` 으로 요청 시마다 생성해 잘못 캐시된 HTML 응답 방지. try/catch로 에러 시에도 최소한 메인 URL만 담은 사이트맵(XML) 반환. |

### 4.3 확인 방법

- 배포 후 브라우저에서 `https://www.qknou.kr/sitemap.xml` 을 열어 **`<urlset>`** 로 시작하는 XML이 보이는지 확인하세요.
- 서치 콘솔에서 사이트맵을 **다시 가져오기** 한 뒤 상태를 확인하세요.

## 5. 요약

| 구분 | 강조 포인트 |
|------|-------------|
| **네이버** | 내부링크(2~3개), 하단 요약·재방문 유도, 시리즈 연재로 체류·신뢰도 확보 |
| **구글** | 출처 명시, H2/H3 구조, 신뢰도 높은 외부링크로 E-E-A·품질 시그널 강화 |
| **공통** | 제목·본문 키워드, 제목 계층, 이미지 설명, 요약·CTA로 전환까지 이어가기 |

이 가이드를 기준으로 페이지별 적용 여부를 점검하면 됩니다.

## 네이버 SEO 기본 가이드 – 항목별 점검

네이버 검색로봇이 사이트를 잘 이해하도록 아래 항목을 지키는 것이 좋습니다.

| 가이드 항목 | 요구 사항 | 현재 상태 |
|-------------|-----------|------------|
| **&lt;title&gt; 요소** | `head` 안에 1개, 정확·고유한 제목 | ✅ 루트 `layout.tsx`에서 메타데이터로 설정. 페이지별 `generateMetadata`로 **고유 제목** 부여(과목·연도·암기/시험모드). |
| **&lt;title&gt; 2개 이상** | 문서당 `title` 1개만 | ✅ Next.js 메타데이터로 1개만 출력. 별도 `<title>` 태그 없음. |
| **동일 제목 다수** | 페이지마다 고유한 `title` | ✅ 홈 / 과목별 연도 / 암기모드 / 시험모드별로 **서로 다른 title** 설정됨. |
| **title 길이** | 너무 긴 제목 지양 | ✅ 템플릿으로 `과목명 \| 방송통신대학교 기출문제 \| 큐노` 등 적정 길이 유지. |
| **&lt;meta name="description"&gt;** | `head` 안에 페이지 요약(여러 문장) | ✅ 루트에 기본 description. 페이지별 **고유 description** (`generateMetadata`) 적용. |
| **동일 description 다수** | 페이지마다 고유한 설명 | ✅ 과목·연도·모드별로 **서로 다른 description** 적용. |
| **&lt;H1&gt; 2개 이상** | 페이지당 H1은 1개만 | ✅ 메인 1개, 연도 선택 1개, 시험모드 1개, 404 1개. 암기모드는 H3만 사용( H1 2개 아님). |
| **Alt 속성** | 콘텐츠 이미지에 `alt` | ✅ 로고, 메인 아이콘, 챗봇 등 `next/image` 사용 시 `alt` 지정됨. |

요약: **title/description 고유화**는 `exam/[subjectId]/year`, `memorize-mode`, `test-mode` 페이지에 `generateMetadata`를 추가해 충족했습니다. 그 외 항목은 기존 구조로 충족됩니다.
